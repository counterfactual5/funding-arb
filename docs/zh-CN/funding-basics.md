# 资金费率基础

费率机制、Delta 中性与策略总览

## 资金费是什么

<!-- id: fb-what -->

永续合约没有到期日，交易所用资金费（funding）机制把永续价格锚定在现货指数附近。每隔一个结算周期（1h / 2h / 4h / 8h），多头与空头之间按持仓名义价值互相支付费用。

- 费率为正：多头支付给空头（永续价格高于指数，市场偏多）
- 费率为负：空头支付给多头（永续价格低于指数，市场偏空）

> ℹ️ 资金费是多空双方互转，交易所不抽成（手续费另计）。套利者的目标：站在收钱的一边，同时用另一条腿对冲价格风险。

## 费率与结算周期

<!-- id: fb-rate -->

各所公布的 rate_pct 是「当前结算周期内」的费率，周期长度不同，不能直接比较绝对值：

| 交易所 | 典型结算周期 |
| --- | --- |
| Binance / OKX / Bybit | 8h |
| Bitget | 2h 或 8h（按合约） |
| Hyperliquid / Lighter / dYdX v4 | 1h |
| EdgeX | 4h（主流合约） |
| Aster | 按合约（常见 8h，读 fundingInfo） |

```text
annual_pct ≈ |rate_pct| × (24 / interval_h) × 365
```

例如：8h 周期的 0.01% 年化约 10.95%；1h 周期的 0.01% 年化约 87.6%。同样的数字，周期越短年化越高。

## 溢价与费率的关系

<!-- id: fb-premium -->

资金费率本质上跟踪「溢价」：标记价（mark）相对指数价（index）的偏离。永续被买得越高于指数，下一期费率越偏正；反之越偏负。

```text
basis_pct = (mark_price − index_price) / index_price × 100%
```

在跨周期配对中，本系统用该基差对下一期 funding 做加权估计（basis blend），详见「跨周期资金费率套利」。

dYdX v4 类似：每分钟采样 impact 价相对 Oracle 的溢价，60 分钟平均后加利率项，每小时整点结算。Indexer 的 nextFundingRate 已是下一小时预测值。

## Delta 中性对冲

<!-- id: fb-delta -->

只做一条腿收资金费会完全暴露在价格波动中。套利的关键是两条腿对冲：一腿收资金费，另一腿抵消价格涨跌。

- 现货多 + 永续空 —— 正向 Cash & Carry
- 借币卖出 + 永续多 —— 反向 Cash & Carry
- 永续多 + 永续空（不同所）—— Pure Futures

> ℹ️ 对冲后价格涨跌基本不影响净值（Delta ≈ 0），收益来源是 funding 减去手续费与借币成本。

## 三种策略总览

<!-- id: fb-strategies -->

| 策略 | 两条腿 | 收益来源 | 适用场景 |
| --- | --- | --- | --- |
| Cash & Carry（同所） | 现货 + 永续，同一所 | 单所费率的绝对值 | 某所费率极端（很正或很负） |
| Unified C&C（跨所） | 现货腿与期货腿拆在不同所 | 最高费率 + 最低成本的组合 | 各所费率 / 手续费 / 借币成本差异大 |
| Pure Futures | 两所永续，一多一空 | 两所费率之差 | 跨所费率分化；DEX 可参与 |

跨周期配对（如 HL 1h vs CEX 8h）是 Pure Futures 的进阶专题，单独成篇。

## 共性风险

<!-- id: fb-risks -->

| 风险 | 说明 | 系统应对 |
| --- | --- | --- |
| 费率翻转 | 入场后费率转向，从收钱变倒贴 | exit 阈值 + watcher 持续监控 |
| 价格错配 | 两腿开仓价偏离，入场即浮亏 | mark_spread 过滤；real_edge 排序 |
| 手续费侵蚀 | 开 + 平共四次 taker | net_edge 预扣开仓费；VIP 费率策略 |
| 强平风险 | 永续腿有杠杆，极端行情可能爆仓 | 保证金健康度监控（margin health） |
| 借币成本浮动 | 反向 C&C 的利息随市场变化 | borrow_per_period 计入边际并持续重估 |
