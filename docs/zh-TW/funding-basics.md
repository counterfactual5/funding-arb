# 資金費率基礎

費率機制、Delta 中性與策略總覽

## 資金費是什麼

<!-- id: fb-what -->

永續合約沒有到期日，交易所用資金費（funding）機制把永續價格錨定在現貨指數附近。每隔一個結算週期（1h / 2h / 4h / 8h），多頭與空頭之間按持倉名義價值互相支付費用。

- 費率為正：多頭支付給空頭（永續價格高於指數，市場偏多）
- 費率為負：空頭支付給多頭（永續價格低於指數，市場偏空）

> ℹ️ 資金費是多空雙方互轉，交易所不抽成（手續費另計）。套利者的目標：站在收錢的一邊，同時用另一條腿對沖價格風險。

## 費率與結算週期

<!-- id: fb-rate -->

各所公佈的 rate_pct 是「當前結算週期內」的費率，週期長度不同，不能直接比較絕對值：

| 交易所 | 典型結算週期 |
| --- | --- |
| Binance / OKX / Bybit | 8h |
| Bitget | 2h 或 8h（按合約） |
| Hyperliquid / Lighter | 1h |
| Aster / EdgeX | 1h ~ 8h（按合約） |

```text
annual_pct ≈ |rate_pct| × (24 / interval_h) × 365
```

例如：8h 週期的 0.01% 年化約 10.95%；1h 週期的 0.01% 年化約 87.6%。同樣的數字，週期越短年化越高。

## 溢價與費率的關係

<!-- id: fb-premium -->

資金費率本質上跟蹤「溢價」：標記價（mark）相對指數價（index）的偏離。永續被買得越高於指數，下一期費率越偏正；反之越偏負。

```text
basis_pct = (mark_price − index_price) / index_price × 100%
```

在跨週期配對中，本系統用該基差對下一期 funding 做加權估計（basis blend），詳見「跨週期資金費率套利」。

## Delta 中性對沖

<!-- id: fb-delta -->

只做一條腿收資金費會完全暴露在價格波動中。套利的關鍵是兩條腿對沖：一腿收資金費，另一腿抵消價格漲跌。

- 現貨多 + 永續空 —— 正向 Cash & Carry
- 借幣賣出 + 永續多 —— 反向 Cash & Carry
- 永續多 + 永續空（不同所）—— Pure Futures

> ℹ️ 對沖後價格漲跌基本不影響淨值（Delta ≈ 0），收益來源是 funding 減去手續費與借幣成本。

## 三種策略總覽

<!-- id: fb-strategies -->

| 策略 | 兩條腿 | 收益來源 | 適用場景 |
| --- | --- | --- | --- |
| Cash & Carry（同所） | 現貨 + 永續，同一所 | 單所費率的絕對值 | 某所費率極端（很正或很負） |
| Unified C&C（跨所） | 現貨腿與期貨腿拆在不同所 | 最高費率 + 最低成本的組合 | 各所費率 / 手續費 / 借幣成本差異大 |
| Pure Futures | 兩所永續，一多一空 | 兩所費率之差 | 跨所費率分化；DEX 可參與 |

跨週期配對（如 HL 1h vs CEX 8h）是 Pure Futures 的進階專題，單獨成篇。

## 共性風險

<!-- id: fb-risks -->

| 風險 | 說明 | 系統應對 |
| --- | --- | --- |
| 費率翻轉 | 入場後費率轉向，從收錢變倒貼 | exit 閾值 + watcher 持續監控 |
| 價格錯配 | 兩腿開倉價偏離，入場即浮虧 | mark_spread 過濾；real_edge 排序 |
| 手續費侵蝕 | 開 + 平共四次 taker | net_edge 預扣開倉費；VIP 費率策略 |
| 強平風險 | 永續腿有槓桿，極端行情可能爆倉 | 保證金健康度監控（margin health） |
| 借幣成本浮動 | 反向 C&C 的利息隨市場變化 | borrow_per_period 計入邊際並持續重估 |
